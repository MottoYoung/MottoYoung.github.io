Transformer初探

MottoYoung Lv1
  2025-07-02 09:39:40 2025-07-02 09:39:40 Created   2025-07-02 10:08:14 2025-07-02 10:08:14 Updated  

  • Encoder

    1.将单词转换为 数字——word embedding

    • token的概念
    • 单词的数字化（每个数字都是从某种维度上描述的空间位置，单词的联系越近，其距离越小） （在该例子中，一个单词四维向量来表示）

    2.融合单词的顺序 positional Encoding(位置编码)

    一种主要的编码方法

    • 如图 这里有四个正余弦曲线，每一个词的x（第几个）相同，用这个x在正余弦上取到四个不同的y值，这个y值叫 positional encoding 与 单词的 word enbeddings相加得到 整个句子的

    3.自注意力机制（self-attention）——计算句子（输入）中每个单词之间的关联程度

    • 如下图中 it应该与 pizza关联

    计算过程：

    1.Query key 的计算

    对于要查询的词求一个query，然后对于所有词都求一个key

    2 相似度计算

    query 点乘 key得到要查询的词和自己以及其他词的相似度（这里去查看余弦相似度，也可以用其他方式算相似度）

    3.计算各单词(包含本身)所占注意力的权重

    • 计算目前单词与输入的句子中的各个单词（包含自身）的 percentage(此处是encoder，算的是输入的，后面的decoder也是这个过程，只不过算的是输出的）
    • 将相似度用softmax转一下（输出为0-1范围） ，作为 对当前单词进行编码时 所有单词占的权重

    4.计算单词的注意力

    • 计算
      给输入的句子中的所有词搞一个新变换得到value ，单词的value乘各自的注意力权重 ，得到目前单词的自注意力值（图中算的是lets’s go 这个句子 中 lets’s 的自注意力值，2.5 -2.1）

    • 迭代
      计算go（其他单词）的时候 只需要计算一下 go（其他单词）的query 不用再计算key，然后用query 点乘 key得到相似度
      然后用各个单词的value 乘 softmax(相似度)的值 计算出 go（下一个单词）的自注意力编码

      注意：

    • 无论输入中有多少个单词，计算 query key value时所用的权重都是一样的

    • 因为计算每个单词的qky时用的权重都一样，所以对于不同单词 其q，k和v计算可以是并行的。（但是每个单词（token）的自注意力值需要用到自己的Q和其他所有的v 只能挨个计算，所以token是逐个生成的）

    5. 残差链接

    得到自注意力值后 建立残差链接

    6.问题 & 拓展

    1.为什么不直接使用positional encoding来代表单词，而是经过qkv之后得到的self-values来代表？（原视频中好像是这样表述的，我不明白为什么问这个，可能是self-attention values与qkv中的v相同？）

    • 自注意力值来自于句子中的所有单词，可以更好地捕捉到不同单词之间的上下文关系
    • 可以将这一套权重看做 注意力单元， 然后用一系列不同权重的注意力单元来捕捉单词之间不同的关系，最初的transformer有8个 这种单元，叫做多头注意力
      不过我很好奇,为什么引入一个v来计算 self-attentions 而不是用 positional encoding，v和self-attention values一定相同吗
    • v与self-attention values 不同 v是构造的比较符合实际语义特征的编码
    • q是为了定位关注点引入的
    • key是为了提供可匹配的语义特征
    1. 多头注意力机制
      采用不同的qkv计算方式（计算qkv时的权重不一样）
    2. 交叉注意力机制（与自注意力相对，自注意力qkv都来自于同一输入)
      Q来自 Decoder kv来自于encoder 用于对其跨语言信息
    3. 结构

    Decoder

    1.word embedding

    首先对 要输出的单词进行word embedding 从EOS开始（这是常见的初始化decode encodeed inputs 进程的方式，有时也有人用 SOS（start of sequence)来初始化这个过程）

    2.positional encoding(用和之前一样的cos sin函数)

    3.计算self attention （和之前类似，但是weights不同）

    4.建立输出和输出之间的注意力机制（encoder-decoder attention)

    • 建立原因：decoder要抓住关键词

    计算过程（用另一套权重）

    1.对decoder的内容（此处为EOS）求query，然后与encoder的输出 点乘，

    1. 得到 每个输入的单词在预测输出的单词时 的贡献度（percentage）

    2. 再进一步得到 encoder-decoder attention的值（就是每个词的value x 占比然后相加，
      只是针对某一个词的）

    3. 将计算出的 encoder-decoder attention 的值 引入残差链接

    4. 残差链接之后的数值 放入全连接层

    5. 计算 每个单词的概率（将1 decoder的 residual connection输入全连接层 计算出四个值，然后丢进softmax，得到对应单词的概率 输出概率最大的单词 （vamos一个词就是 let’s go的意思））

    然后对于 输出的 vamos 继续输入decoder,重复以上过程（
    1. 对 vamos进行 embedding
    2. positional encoding
    3. Q k v计算自注意力，残差链接
    4. 计算 encoder-decoder attention
    5. 得到 EOS 结束（输出EOS之前不会结束）

    [!NOTE]
    计算query key value 的权重和encoder不同,但是decoder里计算每个单词时的权重都相同，也可以像前面一样引入多组权重，形成多头注意力机制

    结束阶段（下一步）

    但是decoder还没有停止 因为其还没有输出 EOS

    继续执行（将上一步得到的输出 vamos 输入decoder）

    1. 将 vamos输入decoder的 embedding 层

    2. 得到word embedding for vamos

    3. add postional encoding

    4. 计算 vamos在输出句子中的自注意力

    5. 添加 残差链接（计算后得到 vamos 的self-attention）

    6. 计算 encoder-decoder attention

    7. 继续建立残差链接（2.6 1.2 与 -1.2 5.5 加起来 得到 1.4 6.7）

    8. 将得到值 输入全连接 层 得到预测的词

    总结

    • word embedding 将单词转变为数字
    • positionanl encoding 追踪单词顺序
    • 自注意力机制 追踪输入或输出的句子中 单词之间的联系
    • encoder-decoder 用于追踪 输入与输出之间的关系，来确保重要单词不丢失
    • 残差链接 让每个单元专注于一块的问题

    补充

    • 原始的transformer 37000多个单词，输入输出很多，每一步之后都归一化（normalization)
    • 相似度上也可以做文章，也可以放缩 （scalaing ） dot product 研究发现 scaling 有助于长难句的编解码
